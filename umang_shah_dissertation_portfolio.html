The Quest for Recovery:
Digital Forensic Tools vs Anti-Forensic Techniques

Overview

Project Summary

The digitalisation of the world has increased the frequency of computer crimes, making digital forensic tools essential in modern investigations. This research conducted an in-depth empirical analysis of open-source digital forensic tools — Autopsy, FTK Imager, Recuva, PhotoRec (QPhotoRec), and Foremost — evaluating their effectiveness at recovering deleted data from USB drives, SD cards, Hard Disk Drives (HDD), and Virtual Hard Disk Drives (VHD) after the application of three anti-forensic deletion techniques: standard Windows deletion, disk formatting, and secure DOD 5220.22M three-pass overwriting.

All experiments were conducted following ACPO Good Practice Guide for Digital Evidence, with full chain-of-custody documentation maintained across all 12 test cases. Tool performance was evaluated under NIST Computer Forensics Tool Testing (CFTT) metrics: number of files recovered, file types recovered, recovery rate, false positive rate, processing speed, reliability, and storage media compatibility.

The Problem

Why This Research Matters

Cybercrime is escalating globally — over 1.5 billion cases were recorded through 2021. In a high-profile example relevant to this research, attackers struck Arnold Clark (UK) with ransomware, encrypting customer data for 2.5 million individuals. As attacks grow more sophisticated, so do the anti-forensic techniques criminals use to erase evidence trails.

Digital forensic investigators face a critical challenge: when evidence is deliberately destroyed using anti-forensic tools, which recovery tools can still surface usable data? Existing research compared tools in isolation or failed to test against real deletion techniques. This project addressed that gap directly — through controlled experimentation that mirrors real-world investigation scenarios.

Research Question: How effective are open-source digital forensic tools at recovering data deleted by three progressively destructive anti-forensic techniques — and which tool performs best in which scenario?

Methodology

Experimental Research Design

The research followed an Experimental Research Model under ACPO Good Practice Guide for Digital Evidence — the UK standard for forensic investigation integrity. Each test case was documented with a full chain of custody including timestamps, investigator name, actions performed, and storage locations.

01
Dataset Creation
A 1.3 GB dataset of 520 files was assembled from NIST CFREDS, Digital Corpa, and curated public sources — spanning 8 file categories: archives, audio, documents, executables, images, text, video, and folders. Custom dataset chosen over NIST images to enable controlled deletion and known-ground-truth recovery comparison.
02
Storage Media Preparation
Data was loaded onto four storage media types: USB Drive (16GB, exFAT), SD Card (32GB, exFAT), HDD partition (1.5GB, NTFS), and VHD (1.5GB, FAT). Each represents a distinct file system and physical medium relevant to forensic investigations.
03
Anti-Forensic Deletion
Three deletion techniques were applied per storage device: (1) Standard Windows Delete + Shift+Delete, (2) Windows Format, (3) File Shredder using DOD 5220.22M 3-pass algorithm — yielding 12 total test cases. All deletions were timestamped in chain-of-custody logs.
04
Evidence Preservation
Write blockers (EaseUS Partition Manager) were applied to all external devices before scanning. Forensically sound disk images were created using FTK Imager (.E01) and OS Forensics (.img, .raw) to preserve evidence integrity per ACPO Principle 2.
05
Tool Execution & Data Collection
Each of the 5 forensic tools was run against every test case — 60 total tool runs. Recovery results were recorded per file type against known original counts. Processing time was measured in minutes and seconds for each run.
06
Analysis Against NIST Metrics
Results evaluated across 7 NIST CFTT metrics: files recovered, file types recovered, recovery rate %, false positive rate, processing time, reliability (consistency across runs), and storage media compatibility.
Tools Evaluated

Five Forensic Tools. One Rigorous Test.

All tools are open-source or free versions, selected to represent the toolkit available to investigators without commercial licensing. Each was evaluated for functional requirements (data recovery options, anti-forensic detection, file carving, metadata analysis) and non-functional requirements (performance, usability, reliability, compatibility).

Autopsy
Full Forensic Suite · Cross-Platform
Open-source GUI forensic platform with deep scan, metadata analysis, file carving, and module-based extensibility. Supports FAT, NTFS, EXT, HFS+, APFS. Comparable in capability to commercial tools.
70.9%
FTK Imager
Forensic Imaging + Analysis · Windows
Industry-standard forensic imaging tool with file signature analysis, hash verification, and registry viewer. Fastest processing time (avg 7.66 sec) but relies on file signatures — vulnerable to formatting. Used by law enforcement globally.
42.6%
Recuva
Consumer + Deep Forensic · Windows
Deep scan with recycle bin and explorer context integration. Highest average recovery rate across all cases. Supports NTFS, FAT. Secure deletion detection via advanced scan modes makes it the most versatile free recovery tool tested.
72.9%
QPhotoRec
File Carver · Cross-Platform GUI
GUI version of PhotoRec — uses cluster-block and sector-by-sector carving to recover individual files (not partitions). Broad file system support (NTFS, exFAT, FAT, EXT). Strong in formatting scenarios; limited in overwriting recovery.
57.2%
Foremost
Header/Footer Carver · Linux CLI
Command-line tool using file header/footer signatures for recovery. Highly customisable file type targeting and multithreaded performance. Lowest average recovery rate (37.4%) due to limited file type support — cannot recover .mp3, .txt files.
37.4%

Key Findings

Eight Findings That Advance the Field

🏆
Recuva & Autopsy Lead Overall
Recuva (72.9%) and Autopsy (70.9%) are the most effective free tools across all scenarios. Investigators should default to both in combination for maximum coverage.
⚠️
FTK Fails on Formatting
FTK Imager achieves 0% recovery on formatted drives — its file-signature-dependent methodology is destroyed by formatting. This is a critical operational blind spot for investigators using FTK alone.
🔒
DOD 5220.22M Confirms Secure Deletion Works
All five tools failed to recover content from DOD-overwritten devices. Enterprises can rely on DOD 5220.22M for compliant data disposal. Directly relevant to GDPR storage limitation principle and DPDP Act secure deletion requirements.
📁
File System Affects Recovery Success
NTFS and FAT file systems preserve metadata artifacts ($MFT, $FAT1, $MBR) that aid reconstruction even after severe deletion. EXFAT recovery relies more heavily on unallocated space carving.
⏱️
Speed vs Depth Trade-off
FTK Imager is fastest (avg 7.66 sec) but least thorough in formatting cases. Autopsy is slowest (avg 21.85 min) but most comprehensive. Investigators must balance urgency against recovery completeness.
🚫
Foremost's File Type Limitation
Foremost cannot recover .mp3, .txt, and several common file types — making it inadequate as a standalone tool despite its customisability. Only suitable as a supplemental carving tool for specific file types.
🔬
Partial Recovery Has Investigative Value
Even in overwriting cases, Autopsy recovered .ZZZ fragments and NTFS system files ($Repair, $TXFLog, $MFT) — confirming file existence and providing reconstruction anchor points for advanced forensic follow-up.
📐
4K Alignment Improves Outcomes
VHD devices with 4K alignment showed higher recovery success — 4096-byte block storage minimises fragmentation, increasing the probability of intact file segment recovery. Recommendation for forensic lab storage configuration.

Impact & Future Directions

What This Research Opens Up

The study identified several directions for future research in digital forensics tool development and evaluation:

AI/ML-Enhanced Recovery Algorithms — Traditional carving tools use static file signatures. Future work could apply neural network-based pattern recognition to identify partial file fragments from overwritten data — potentially recovering content currently deemed unrecoverable by all five tools tested.
Cloud & Large-Scale Forensics — Processing time and storage constraints limited this study to 1.3GB datasets. Cloud computing infrastructure could enable forensic analysis of multi-terabyte devices — solving the investigator's real-world challenge with enterprise storage media.
Quantum & DNA-Based Recovery — Emerging technologies including quantum computing and DNA data storage may enable recovery beyond what is currently theoretically possible — including reconstruction from multiply-overwritten storage sectors.
Commercial Tool Benchmarking — This research was constrained to open-source tools. Future work extending the methodology to EnCase, Cellebrite, and X-Ways Forensics would provide a complete industry picture — and determine whether commercial licensing is justified for specific anti-forensic scenarios.

The findings of this research have direct applicability to enterprise data security policy (specifically secure data disposal and breach investigation readiness), digital forensics training programmes, and law enforcement tool procurement guidance.
